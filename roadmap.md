# Tapir.Group — Nuxt 4 Test Task (SSR + TypeScript + SCSS + BEM)

Nuxt-приложение по макету: главная + каталог товаров с SSR и пагинацией “Показать ещё”.

## Стек
- Nuxt 4
- TypeScript
- SCSS
- SSR
- Grid/Flex (адаптив)
- БЭМ-нейминг классов

## API
- `https://test-task-api.tapir.ws/products`
- Примеры:
  - `https://test-task-api.tapir.ws/products?page=1`
  - `https://test-task-api.tapir.ws/products?page=2&limit=10`

## Реализовано по ТЗ

### Header
- Все ссылки заглушки: `href="#"`
- Бургер-кнопка без функционала
- Мобильное меню не реализуется

### Главная
- Кнопка “В каталог” ведёт на `/catalog`

### Каталог
- Вся карточка товара — ссылка-заглушка `href="#"`
- Кнопка “Добавить в избранное” есть визуально, действий не выполняет
- Цена “x2” рассчитывается как `ceil(price / 2)`
- Первая загрузка списка товаров на `/catalog` выполняется с SSR (серверный рендер обязателен)
- “Показать ещё”:
  - при клике кнопка скрывается, показывается индикатор “Загрузка...”
  - при успехе индикатор скрывается, товары добавляются в конец списка
  - если есть следующая страница — снова показывается кнопка “Показать ещё”
  - при сетевой ошибке:
    - индикатор скрывается
    - под списком отображается сообщение “Произошла ошибка, попробуйте позже”
    - кнопка “Повторить” делает повторный запрос

### Footer
- Все ссылки заглушки: `href="#"`
